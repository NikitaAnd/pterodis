# pterodis

Discord-бот для управления серверами в Pterodactyl Panel.

## Что умеет бот

- Управление питанием серверов: запуск, остановка, перезапуск.
- Получение статуса сервера (CPU, RAM, диск, RX/TX, uptime).
- Быстрые кнопки управления прямо в ответе команды `/status`.
- Гибкая система прав:
  - `SUPER_USERS` (полный доступ),
  - назначение/удаление доступа на конкретные сервера.
- Автодополнение серверов с учетом прав пользователя.
- Кэш серверов с TTL + ручное обновление кэша (`/syncservers`).
- Хранение прав доступа в JSON (с миграцией из legacy `user_access.pkl`).

## Требования

- Python 3.9+
- `disnake`
- `py-dactyl`

## Установка

```bash
pip install disnake py-dactyl
```

## Настройка окружения

Перед запуском задайте переменные окружения:

- `DISCORD_TOKEN` — токен вашего Discord-бота.
- `PTERODACTYL_URL` — URL панели Pterodactyl.
- `PTERODACTYL_API_KEY` — API-ключ панели.
- `SUPER_USERS` — список ID супер-пользователей через запятую (например: `123,456`).
- `USER_ACCESS_FILE` *(опционально)* — путь к JSON с доступами (по умолчанию `user_access.json`).
- `SERVER_CACHE_TTL_SEC` *(опционально)* — TTL кэша серверов в секундах (по умолчанию `60`).
- `LOG_LEVEL` *(опционально)* — уровень логирования (`INFO`, `DEBUG` и т.д.).

Пример:

```bash
export DISCORD_TOKEN="your_discord_token"
export PTERODACTYL_URL="https://panel.example.com"
export PTERODACTYL_API_KEY="ptla_your_api_key"
export SUPER_USERS="123456789012345678,987654321098765432"
python pterodis.py
```

## Команды

- `/start` — запуск сервера.
- `/stop` — остановка сервера.
- `/restart` — перезапуск сервера.
- `/status` — состояние сервера + кнопки управления.
- `/servers` — список доступных серверов.
- `/myaccess` — ваши текущие доступы.
- `/adduser` — добавить пользователя на сервер.
- `/deluser` — удалить пользователя с сервера.
- `/syncservers` — принудительно обновить кэш серверов (только `SUPER_USERS`).

## Безопасность

- Не храните токены/API-ключи в исходном коде.
- Используйте переменные окружения.
- Выдавайте `SUPER_USERS` только доверенным администраторам.
